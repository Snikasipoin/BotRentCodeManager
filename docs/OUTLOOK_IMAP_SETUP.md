# Outlook / Hotmail IMAP для получения Steam Guard и FACEIT кодов

Этот документ про практическую настройку почт Outlook / Hotmail для работы бота.

## Короткий вывод

Для Outlook / Hotmail самый рабочий вариант сегодня:
- включить двухфакторную аутентификацию;
- создать пароль приложения;
- использовать IMAP с этим паролем приложения.

Параметры обычно такие:
```env
IMAP host: imap-mail.outlook.com
IMAP port: 993
SSL: enabled
```

## Что нужно сделать для одной почты

### 1. Включить двухэтапную проверку Microsoft
Без нее пароль приложения чаще всего недоступен.

### 2. Создать пароль приложения
Обычно путь такой:
- аккаунт Microsoft
- Безопасность
- Дополнительные параметры безопасности
- Пароли приложений
- Создать новый пароль приложения

Сохраните этот пароль сразу. Повторно его Microsoft обычно не показывает.

### 3. Использовать в боте именно пароль приложения
Не обычный пароль от почты, а пароль приложения.

В бота нужно записывать:
- `email`
- `email_password` = пароль приложения
- `email_imap_host` = `imap-mail.outlook.com`
- `email_imap_port` = `993`

## Почему обычный пароль часто не работает

Outlook / Hotmail регулярно режет "небезопасные" IMAP-входы. Даже если логин и пароль правильные, обычный пароль может не пускать.

Поэтому для бота лучше считать правилом:
- Outlook = только через app password.

## Когда аккаунтов много и это неудобно

Да, при большом количестве Outlook-почт это действительно неудобно. Основная боль:
- на каждой почте нужно отдельно включать 2FA;
- на каждой почте нужно отдельно создавать app password;
- Microsoft может дополнительно просить подтверждения активности;
- часть старых/подозрительных аккаунтов может попасть под ограничения IMAP.

## Что облегчает работу на практике

### Вариант 1. Менеджер паролей + таблица учета
Самый простой и реалистичный вариант для большого числа почт.

Рекомендую хранить таблицу:
- email
- обычный пароль
- app password
- 2FA включена / нет
- IMAP проверен / нет
- примечания

Это не автоматизирует Microsoft, но сильно снижает хаос.

### Вариант 2. Перевести аккаунты на почты своего домена
Это лучший вариант в долгую.

Если аккаунты Steam / Faceit можно перевести на адреса под вашим доменом, то удобнее использовать провайдера, где:
- IMAP работает стабильно;
- можно централизованно управлять ящиками;
- не нужно вручную бороться с Microsoft-защитами на каждом аккаунте.

Практически это сильно удобнее, чем десятки consumer Outlook-почт.

### Вариант 3. Microsoft 365 + Graph API
Это уже более взрослый enterprise-путь.

Плюсы:
- лучше, чем IMAP, если у вас контролируемая инфраструктура;
- можно строить более надежную интеграцию.

Минусы:
- сложнее настройка;
- для обычных разрозненных Outlook / Hotmail аккаунтов не всегда удобно;
- потребует отдельной разработки OAuth / Graph flow.

## Есть ли инструмент, который полностью снимает боль с Outlook?

Коротко: нет, для обычных личных Outlook / Hotmail аккаунтов "волшебной кнопки" нет.

Самые практичные решения:
- app passwords + учет в password manager;
- перенос на свой почтовый домен;
- в будущем переход с IMAP на Microsoft Graph API, если захотите делать enterprise-версию.

## Что я рекомендую именно вам

Если аккаунтов уже много, то я бы рекомендовал так:

### Краткосрочно
- оставить поддержку Outlook IMAP через app passwords;
- завести таблицу учета почт;
- на каждую новую почту сразу делать 2FA + app password + тест IMAP.

### Среднесрочно
- новые аккаунты переводить на почты, которыми вы управляете централизованно;
- постепенно уменьшать долю Outlook consumer mailbox.

### Долгосрочно
- добавить в проект отдельный провайдер для Microsoft Graph API;
- оставить IMAP как fallback.

## Что проверять, если код не приходит

1. Письмо реально пришло в почту.
2. App password введен правильно.
3. IMAP host = `imap-mail.outlook.com`.
4. IMAP port = `993`.
5. На аккаунте включена двухэтапная проверка.
6. Microsoft не запросил дополнительное подтверждение входа.
7. Письмо не попало в Spam / Junk.

## Практический совет для массовой работы

Если вы понимаете, что Outlook становится бутылочным горлышком, лучше не бороться с ним бесконечно, а заранее планировать миграцию на более удобную почтовую схему.

Для бота текущий IMAP-путь подойдет как рабочий старт, но для большого масштаба лучшая стратегия — почта под вашим контролем.
